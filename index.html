Overview of ContextRL. We augment GRPO with a context-awareness loss built from contrastive context pairs. For each query–answer pair \((Q, A)\), an anchor context \(C^+\) supports the answer while a confounder \(C^-\) is superficially similar but inconsistent with it. Given the answer, the auxiliary objective trains the policy to select the supporting context over the confounder, improving context-aware reasoning in both coding and multimodal settings. Key finding: ContextRL improves coding & multimodal performance; SFT-only does not.

Introduction

Large language models (LLMs) have progressed into general-purpose reasoning engines capable of operating over rich, long-form contexts — including agentic systems that interleave reasoning with tool use over extended horizons, and multimodal models that combine fine-grained visual perception with textual reasoning. As these systems scale to increasingly complex tasks, their performance depends not only on reasoning ability, but also on grounding decisions in sparse yet decisive contextual evidence: an early observation in a long agent trajectory, a single line in a tool trace, or a subtle visual detail in a dense image.

When such evidence is overlooked, models make locally plausible but context-inconsistent decisions. We call this failure mode context unawareness: the relevant information is present in the context, yet the model's prediction is not grounded in it.

Figure 1: Context unawareness manifests across both agentic and multimodal systems. Left: In an agentic code-editing setting, the model has access to the relevant source file but fails to maintain consistency across edits, removing the definition of variable i that is subsequently referenced — causing a runtime error. Right: In a multimodal reasoning setting, the model fails to correctly ground its answer in the visual evidence. Although \(g(x)\) approaches \(y\!=\!3\) as \(x \to -1\), the model misreads the value as \(2\) and produces an incorrect prediction.

To quantify this failure mode, we introduce a controlled contrastive context probe and find a roughly 40-point gap between proprietary and widely-used open-source models on context selection accuracy — despite open-source models being competitive on standard benchmarks.

Motivated by this, we propose ContextRL (Context-Aware Reinforcement Learning), a post-training method that augments RL with an explicit context-selection auxiliary objective. We construct contrastive context data in two domains: for coding agents, trajectories serve as contexts, yielding 1k pairs built via condition filtering; for multimodal reasoning, images serve as contexts, yielding 7k pairs via generative editing and similarity search. ContextRL is incorporated into standard GRPO post-training via a logit-level contrastive loss that favors the context aligned with the ground-truth answer.

The Contrastive Context Probe

To quantify context unawareness, we construct a controlled contrastive context probe: 200 contrastive pairs from agentic trajectories and 200 from visual question answering (VQA) images. Each example presents the model with a question, a candidate answer, and two closely matched contexts that support different answers — the model must select the context that justifies the candidate answer.

This simple test reveals a ~40-point gap between proprietary models (GPT-4.1, Claude Opus 4.7) and widely-used open-source models. Notably, strong open-source models such as Qwen3-VL 8B and Qwen3.5 9B perform near random choice on this probe, despite their competitive performance on standard benchmarks. These results suggest that strong benchmark performance can obscure failures in context grounding.

Key Insight: Models that score highly on standard benchmarks may still fail to identify the specific evidence that supports a given answer. Context unawareness is a distinct capability gap that standard accuracy-based evaluations do not capture.

Contrastive Context Pairs Construction

We construct contrastive context pairs in both agentic and multimodal domains. We use pipelines specifically designed to yield pairs that are nearly identical at the surface level, differing only in a small but semantically decisive region. This ensures that selecting the correct context requires genuine understanding of the evidence rather than exploitation of surface statistics.

Figure 4: Contrastive context pairs construction pipeline. Left: Step-by-step filtering to mine contrastive trajectory pairs for the agentic setting. Right: Generative editing and similarity-based retrieval to mine contrastive image pairs for the multimodal setting. A concrete \((C^+, C^-)\) example is shown alongside each method.

Agentic: Trajectory Pair Search

We construct contrastive trajectory pairs from 66k trajectories built with SWE-smith. Our pipeline applies a cascade of four increasingly restrictive filters: candidate trajectories must (i) originate from the same repository and commit, (ii) modify the same file, (iii) target the same function or class, and (iv) correspond to distinct but semantically related issues. Patch contents inside edit commands are masked with <PATCH_MASKED> to prevent trivial leakage.

We apply automatic verification with GPT-5 to explicitly screen for and reject pairs with superficial shortcut cues or ambiguous labels. This cascade is deliberately aggressive: only 1k high-quality contrastive trajectory pairs survive.

1k agentic pairs

Multimodal: Image Pair Mining

We collect image, question, and answer triplets from five domains: charts, geometry, non-geometric math, science diagrams, and natural images, covering diverse patterns of visual grounding. Two strategies construct \((C^+, C^-)\) pairs:

Generative editing for natural images: GPT-5 proposes an edit instruction, Nano Banana 2 applies it, and a verifier rejects editing artifacts (blur, warping, implausible lighting) and global restyling. Yields ~700 pairs from 2k candidates.

Similarity-based retrieval for structured images: images are embedded with Qwen3-VL-Embedding-8B; pairs with cosine similarity ≥ 0.85 but different answers are retrieved. GPT-5 filters unrelated pairs. Yields 6,300 pairs from 200k candidates.

7k multimodal pairs

ContextRL

We introduce Context-Aware Reinforcement Learning (ContextRL), a training framework that explicitly encourages models to ground their predictions in the provided context. Our key idea is to augment standard RL post-training with a context-selection signal: instead of only rewarding correct outputs, we additionally train the model to identify which context supports a given answer. Using the contrastive context pairs \((C^+, C^-)\) constructed above, we incorporate them into standard GRPO post-training via a logit-level contrastive loss.

Contrastive Context Pairs

Each pair is associated with a query \(Q\) and an anchor answer \(A\), where \(C^+\) is the supporting context for \(A\), and \(C^-\) is a minimally perturbed but confounding alternative that instead supports a different answer. The context takes the form of an agent trajectory \(\tau\) for agentic coding, or an image \(I\) for multimodal tasks.

Context-Awareness Loss

Given a contrastive instance \(z = (Q, A, C^+, C^-)\), the model is prompted to choose between \(C^+\) and \(C^-\) as a two-way multiple choice. Let \(\ell_\theta^+(z)\) and \(\ell_\theta^-(z)\) be the next-token logits for the option-letter tokens \(t^+\) and \(t^-\) at the answer position, computed by teacher forcing. We define the margin \(\Delta_\theta(z) = \ell_\theta^+(z) - \ell_\theta^-(z)\) and optimize:

$$\mathcal{L}_{\mathrm{CA}}(z;\,\theta)\;=\;-\log\,\sigma\!\left(\mathrm{clip}\!\left(\Delta_\theta(z),\;-c,\;c\right)\right)$$

where \(\sigma(\cdot)\) is the sigmoid function and \(c > 0\) controls margin clipping. This objective encourages correct context selection while preventing large margins from dominating training. The loss is modality-agnostic and applies uniformly to agent trajectories and images.

Joint Training Objective

Let \(\mathcal{D}_{\mathrm{RL}}\) denote the task data and \(\mathcal{D}_{\mathrm{CA}}\) the contrastive dataset. We jointly optimize:

$$\mathcal{L}(\theta)\;=\;\mathbb{E}_{x\,\sim\,\mathcal{D}_{\mathrm{RL}}}\!\left[\mathcal{L}_{\mathrm{GRPO}}(x;\theta)\right] \;+\;\lambda\;\mathbb{E}_{z\,\sim\,\mathcal{D}_{\mathrm{CA}}}\!\left[\mathcal{L}_{\mathrm{CA}}(z;\theta)\right]$$

where \(\lambda > 0\) balances task performance and context awareness. GRPO optimizes for correct outputs, while \(\mathcal{L}_{\mathrm{CA}}\) enforces alignment between outputs and their supporting context. We apply the same formulation across both agentic and multimodal settings.

Analysis: Why Data Augmentation Fails

To isolate whether the improvement arises from the contrastive data itself or from the training objective used to consume it, we compare against two natural baselines that utilize the same contrastive data through standard augmentation:

  • DA-SFT: Supervised fine-tuning on the contrastive data (cross-entropy to predict the correct context), then standard GRPO on the task data.
  • DA-RL: Contrastive examples mixed directly into the RL training stream, treated as a binary decision with reward 1 if the model selects the correct context and 0 otherwise.

Figure 5: Context-selection accuracy versus end-task performance. The \(x\)-axis denotes selection accuracy and the \(y\)-axis denotes end-task performance. Top-right is optimal. DA-SFT achieves high selection accuracy but collapses on the \(y\)-axis. DA-RL fails to learn discrimination at all. ContextRL is the only method that couples high context awareness with consistent end-task improvements across all four model configurations.

Three clear patterns emerge from the analysis:

  1. Outcome-only RL fails to learn context selection. The RL baseline stays near the base-model cluster, showing that final-task rewards alone provide little signal for distinguishing context-grounded reasoning from shortcuts.
  2. DA-SFT learns context selection but harms the policy. DA-SFT achieves the highest selection accuracy in nearly every setting, but SFT on short-answer examples shifts the model distribution away from the long-form, multi-turn patterns the agent scaffold requires — catastrophic in agentic settings, and largely ineffective in multimodal ones.
  3. Context selection skill alone is insufficient. Both DA-SFT and ContextRL push selection accuracy to 85–93%, but only ContextRL consistently improves downstream task performance. The model must acquire context-selection capability without disrupting the policy behaviors required for the original task distribution. The contrastive data alone does not achieve this — the key is the objective.

Why ContextRL Avoids Both Failure Modes

The key factor is not the contrastive data alone, but how the signal is integrated into training. ContextRL incorporates the context-awareness loss as a bounded auxiliary objective inside the on-policy GRPO loop. Two properties are critical:

(a) Updates Remain Constrained

The GRPO importance-ratio clipping and KL regularization keep the policy close to the reference model \(\pi_\mathrm{ref}\), while the clipped margin objective suppresses auxiliary gradients once \(C^+\) and \(C^-\) are well separated. Together, these preserve the original policy distribution and prevent the catastrophic forgetting observed in DA-SFT.

(b) The Auxiliary Signal is Dense

Unlike DA-RL's sparse 0/1 rewards, \(\mathcal{L}_{\mathrm{CA}}\) directly supervises the relative preference between \(C^+\) and \(C^-\) on every contrastive example. Meaningful gradients are therefore produced even when the current policy would rarely sample the correct context — precisely the regime where outcome-only RL struggles.
